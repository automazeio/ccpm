# Task: Ollama Client Library

## Description
Create a shell function library for interacting with the Ollama HTTP API. This foundational library will provide core functions for health checking, listing available models, and generating code responses. All functions must include robust error handling with clear, actionable error messages.

## Acceptance Criteria
- [x] ollama_health_check() function returns status (0 for success, non-zero for failure)
- [x] ollama_list_models() function lists available models in JSON format
- [x] ollama_generate() function sends prompts and receives streaming responses
- [x] Error handling for connection failures with timeout detection
- [x] Timeout handling with configurable timeout values
- [x] All functions have clear error messages with remediation steps
- [x] Functions support custom endpoint configuration via environment variables
- [x] HTTP response codes properly interpreted and reported

## Technical Details

### Implementation Approach
- Create `ccpm/lib/ollama-client.sh` as a sourced library
- Use curl for HTTP requests with appropriate error handling
- Default Ollama endpoint: http://localhost:11434
- Support OLLAMA_ENDPOINT environment variable override
- Implement streaming response handling for real-time feedback

### Key Considerations
- Ollama API uses HTTP POST with JSON payloads
- Health check endpoint: GET /api/tags (lists models)
- Generate endpoint: POST /api/generate with streaming
- Handle both connection errors and API errors differently
- Provide clear diagnostics when Ollama is not running
- Consider timeout values for different operations (health: 5s, generate: 300s)

### Files Affected
- `ccpm/lib/ollama-client.sh` (new file)

### API Reference
```bash
# Health Check
ollama_health_check()
# Returns: 0 if Ollama is reachable, 1 otherwise
# Stdout: Success/failure message

# List Models
ollama_list_models()
# Returns: 0 on success, 1 on failure
# Stdout: JSON array of model objects

# Generate Code
ollama_generate(model, prompt, [options])
# Parameters:
#   - model: Model name (e.g., "codellama:7b")
#   - prompt: Text prompt for generation
#   - options: Optional JSON string with parameters
# Returns: 0 on success, 1 on failure
# Stdout: Streaming response chunks
```
