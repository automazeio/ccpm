# Task: Review Iteration Controller

## Description
Orchestrate the review → feedback → regenerate loop between the local code generator (Ollama) and Claude code reviewer. This controller manages up to 3 review iterations, handles feedback loops, and implements quality gate logic.

## Acceptance Criteria
- [ ] Orchestrates up to 3 review iterations
- [ ] Calls local-code-generator agent
- [ ] Passes output to claude-code-reviewer agent
- [ ] Handles feedback loop: regenerate if issues found
- [ ] User can override and accept at any iteration
- [ ] Logs all iterations for debugging
- [ ] Quality gate logic (approve/iterate/fail)

## Technical Details

### Implementation Approach
- Create orchestration script at `ccpm/lib/review-loop.sh`
- Implement state machine for iteration management:
  1. Generate code (call local-code-generator)
  2. Review code (call claude-code-reviewer)
  3. Parse review result (approve/iterate/fail)
  4. Handle iteration (regenerate with feedback or exit)
- Support user intervention at any iteration
- Log all iterations with timestamps and decisions

### Key Considerations
- Maximum 3 iterations to prevent infinite loops
- Clear exit conditions (approve, fail, max iterations)
- Preserve all iteration artifacts for debugging
- User override mechanism (accept despite issues)
- Feedback must be passed effectively to regeneration
- Handle edge cases (agent failures, malformed output)
