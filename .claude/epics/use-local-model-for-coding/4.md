# Task: Task Routing Hook

## Description
Create a pre-tool-use hook that intercepts Task tool invocations and automatically delegates to either Ollama (for code generation) or Claude (for planning/review). This enables transparent routing based on task content analysis.

## Acceptance Criteria
- [ ] Intercepts Task tool invocations
- [ ] Analyzes task description for code generation keywords
- [ ] Routes to Ollama agent for code generation tasks
- [ ] Routes to Claude for planning/review tasks
- [ ] Logs routing decisions transparently
- [ ] Respects local_llm.enabled config setting

## Technical Details

### Implementation Approach
- Create hook script at `ccpm/hooks/local-llm-route.sh`
- Implement task analysis logic:
  - Parse task description
  - Match against routing rules (from Task 007)
  - Determine target (Ollama vs Claude)
- Call appropriate agent based on routing decision
- Log routing decision with reasoning
- Honor `local_llm.enabled` config flag

### Key Considerations
- Hook must be fast (no noticeable latency)
- Routing logic should be extensible
- Clear logging for debugging routing decisions
- Fallback to Claude if routing is uncertain
- Respect user config and preferences
- Handle cases where local model is unavailable
- Transparent to user unless routing fails
