# Task: Configuration & Health Check Command

## Description
Extend the CCPM settings.json configuration file with a local_llm section for Ollama configuration, and create a user-facing health check command that tests connectivity and provides setup guidance. This provides the configuration foundation and diagnostic tooling for the local LLM feature.

## Acceptance Criteria
- [ ] settings.json extended with local_llm configuration section
- [ ] Default Ollama endpoint and model configured
- [ ] Environment variable support (OLLAMA_ENDPOINT, OLLAMA_MODEL) overrides settings
- [ ] /pm:llm:health-check command created and functional
- [ ] Health check tests Ollama connectivity
- [ ] Health check verifies configured model availability
- [ ] Clear diagnostics output (connection status, available models, configuration)
- [ ] Setup guidance provided when Ollama not detected
- [ ] Exit codes: 0 for healthy, 1 for unhealthy

## Technical Details

### Implementation Approach
- Extend `.claude/settings.json` with new local_llm section
- Create `ccpm/commands/llm/health-check.sh` command file
- Source ollama-client library functions (when Task 001 complete)
- Use structured output format for diagnostics
- Provide actionable next steps based on health check results

### Configuration Schema
```json
{
  "local_llm": {
    "enabled": false,
    "provider": "ollama",
    "endpoint": "http://localhost:11434",
    "model": "codellama:7b",
    "timeout": 300,
    "options": {
      "temperature": 0.7,
      "top_p": 0.9
    }
  }
}
```

### Command Output Format
```
Ollama Health Check
==================
Connection: OK (http://localhost:11434)
Configured Model: codellama:7b
Model Available: YES
Response Time: 45ms

Available Models:
  - codellama:7b (recommended for code)
  - codellama:13b
  - llama2:7b

Status: HEALTHY
```

### Files Affected
- `.claude/settings.json` (modified)
- `ccpm/commands/llm/health-check.sh` (new file)

### Key Considerations
- Command should work standalone for Task 002 completion
- Full integration with ollama-client.sh happens after Task 001
- Provide helpful error messages for common issues:
  - Ollama not installed
  - Ollama not running
  - Model not pulled
  - Network/firewall issues
- Include installation/setup instructions in error messages
