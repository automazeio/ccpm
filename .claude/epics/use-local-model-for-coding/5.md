# Task: Integration Testing

## Description
Comprehensive integration testing of the complete workflow including routing decisions, local LLM execution, and review loop iterations. Validate that all components work together correctly and handle error scenarios gracefully.

## Acceptance Criteria
- [ ] Test /pm:epic-start with code generation tasks
- [ ] Verify routing works correctly (logs show correct model delegation)
- [ ] Validate review loop catches issues and iterates successfully
- [ ] Test error scenarios: Ollama down, model missing, request timeout
- [ ] Test fallback to Claude API when local LLM fails
- [ ] All tests pass with clear, documented results
- [ ] Document any issues found with severity and workarounds
- [ ] Performance metrics collected (latency, token usage, iterations)

## Technical Details

### Implementation Approach
1. **Setup Test Environment**
   - Configure test CCPM workspace
   - Set up Ollama with qwen2.5-coder:32b
   - Prepare test epics with various task types
   - Configure logging for detailed tracing

2. **Test Scenarios**
   - **Happy Path**: Code generation task routed to local LLM, passes review
   - **Review Iteration**: Code needs 2-3 iterations to meet quality standards
   - **Mixed Epic**: Some tasks local, some Claude API
   - **Error Recovery**: Ollama crashes mid-task, falls back to Claude
   - **Model Missing**: Requested model not available, uses fallback
   - **Timeout**: Long-running task hits timeout, handled gracefully

3. **Validation Steps**
   - Check routing logs for correct decisions
   - Verify review feedback is actionable
   - Confirm iterations improve code quality
   - Measure cost savings vs baseline
   - Validate error messages are clear

### Key Considerations
- Test with realistic CCPM workflows (epics, tasks, reviews)
- Capture all logs for debugging
- Measure both success and failure paths
- Document performance characteristics
- Test edge cases: very large files, complex requirements, ambiguous tasks
