# Task: Local Code Generator Agent

## Description
Create a markdown-based agent definition that uses the Ollama client library to generate code locally. This agent will follow CCPM's agent pattern, constructing prompts with file context and requirements, streaming responses for user visibility, and returning structured code output. This is the primary user-facing feature for local code generation.

## Acceptance Criteria
- [ ] Markdown agent file created following CCPM agent pattern
- [ ] Agent constructs prompts with file context and requirements
- [ ] Uses ollama_generate() function from Task 001
- [ ] Streams responses in real-time for user visibility
- [ ] Returns generated code in structured format (with file paths)
- [ ] Handles generation errors gracefully with fallback messages
- [ ] Supports multi-file generation scenarios
- [ ] Respects configured model and options from settings.json
- [ ] Provides progress indicators during generation
- [ ] Validates Ollama health before attempting generation

## Technical Details

### Implementation Approach
- Create `ccpm/agents/local-code-generator.md` following CCPM conventions
- Agent sources ollama-client.sh library
- Reads configuration from settings.json
- Constructs context-aware prompts including:
  - User requirements
  - Existing file contents (if modifying)
  - Project structure context
  - Coding standards
- Streams output with progress indicators
- Parses generated code from model response
- Returns structured results for CCPM to apply

### Prompt Engineering Strategy
```markdown
System: You are a code generation assistant. Generate clean, working code.

Context:
- Project: [project_name]
- Language: [detected_language]
- Task: [user_requirement]

Existing Code:
[file_contents if applicable]

Requirements:
[user_requirements]

Generate the code with clear separation between files.
Use this format:
FILE: path/to/file.ext
```code
...
```
```

### Agent Structure
```markdown
# Local Code Generator Agent

## Purpose
Generate code using local Ollama models for privacy and offline work.

## Capabilities
- Generate new files
- Modify existing files
- Multi-file generation
- Context-aware suggestions

## Usage
/pm:agent local-code-generator [requirements]

## Implementation
[Shell script that uses ollama_generate()]
```

### Files Affected
- `ccpm/agents/local-code-generator.md` (new file)

### Key Considerations
- Pre-check Ollama health before generation attempts
- Handle streaming output with visual feedback (dots, spinners)
- Parse model output to extract code blocks
- Support both single and multi-file generation
- Handle incomplete generations (token limits, timeouts)
- Provide retry mechanism for failed generations
- Cache prompt templates for consistency
- Consider token limits of different models
- Implement graceful degradation if streaming fails

### Error Scenarios
- Ollama not running: Direct to health check command
- Model not available: Suggest model pull command
- Generation timeout: Offer to retry or use smaller model
- Parsing failures: Return raw output with warning
- Token limit exceeded: Suggest breaking task into smaller pieces
